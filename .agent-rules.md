# PropIntel Agent Rules

Every agent working on PropIntel must follow these rules exactly.

## Stack
- Python 3.9 on macOS + Render Linux (Ubuntu)
- Flask backend — entry point: `backend/app.py`
- Render.com auto-deploys on push to `main`
- GitHub repo: `masecfc03-ui/propintel`
- Frontend: static HTML/JS in repo root (`index.html`, `report.html`, `success.html`)

## Critical — Python 3.9 Compatibility
- NO `str | None` type hints — use `Optional[str]` from `typing` module
- NO nested f-strings with backslashes
- NO walrus operator (`:=`) unless absolutely necessary
- NO `match` statements (Python 3.10+)
- Use `from typing import Optional, Dict, List, Tuple` for type hints

## Critical — Never Touch These
- `backend/.env` — production secrets, never read or modify
- `backend/data/` — production database files
- `backend/reports_cache/` — live report cache
- `.github/workflows/ci.yml` — unless explicitly asked to modify CI

## Before Every Push
Run this exact command from `backend/`:
```bash
python3 -c "import app; print('import OK')"
```
If it fails, fix the import error before pushing.

Also run:
```bash
find backend -name "*.py" | xargs python3 -m py_compile && echo "syntax OK"
```

## PDF Report Standard
- **13 pages minimum — never remove sections**
- `pdf_builder.py` generate_pdf_bytes(report) must always produce a multi-page PDF
- Adding data = more pages. Removing data = NEVER acceptable.
- All ATTOM sections show pending box when data unavailable — not errors

## Pipeline Key Names (verified against live output)
When accessing report dict keys, use EXACTLY these:
- `parcel.absentee_owner` (not `.absentee`)
- `parcel.out_of_state_owner`
- `parcel.property_address` (not `.address`)
- `parcel.assessed_land` (not `.land_value`)
- `parcel.assessed_improvement`
- `parcel.owner_mailing` (full string)
- `flood.description`
- `flood.firm_panel`
- `motivation.score`, `motivation.tier`, `motivation.indicators`
- `market_estimate.market_low`, `market_estimate.market_high`
- `market_estimate.range_fmt`

Always verify against live pipeline output before accessing new keys.

## Admin Authentication
- Admin key: `propintel-mason-2026`
- Auth via `X-Admin-Key` header OR `?admin_key=` query param
- NOT via request body `key` field
- `ADMIN_KEY` env var overrides on Render

## API Structure
- `POST /api/analyze` — main pipeline endpoint (rate limited: 10/hr/IP)
- `POST /api/admin/test-email` — send test report email
- `POST /api/admin/generate-token` — run pipeline, return URL (no email)
- `POST /api/admin/bulk-generate` — batch up to 20 addresses
- `GET /api/reports/<token>` — fetch report by token
- `GET /api/health` — service health check
- `GET /api/stats` — admin stats (requires auth)
- `POST /api/webhook` — Stripe webhook (do not modify)

## Code Style
- Add to existing files — don't rewrite unless explicitly asked
- Keep functions small and focused
- Log errors with `log.error()` not `print()`
- All new endpoints need docstrings
- No unused imports

## PR Requirements
1. Test the feature works before opening PR
2. Include exact test command in PR description
3. Link the GitHub issue: `Closes #N`
4. Conventional commit format: `feat:`, `fix:`, `chore:`, `docs:`
5. Keep changes focused — one issue per PR

## Testing
After any backend change:
```bash
cd backend && python3 test_paid_flow.py
```
Should show 6/6 passing. If not, fix it before pushing.

## Environment Variables (Render)
Required: `REGRID_API_KEY`, `STRIPE_LIVE_SECRET_KEY`, `STRIPE_WEBHOOK_SECRET`, `MAILGUN_API_KEY`, `ADMIN_KEY`
Optional: `ATTOM_API_KEY`, `ZILLOW_RAPIDAPI_KEY`, `PDL_API_KEY`, `DATAZAPP_API_KEY`, `DATABASE_URL`
